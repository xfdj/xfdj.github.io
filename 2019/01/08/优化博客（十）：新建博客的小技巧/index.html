    优化博客（十）：新建博客的小技巧

    发表于 2019-01-08 更新于 2022-05-20 本文字数： 1.5k 阅读时长 ≈ 2 分钟

    每次新建博客，输入hexo new命令后，都要到source/_posts目录下打开创建的md文件，时间一长吧，就有点烦了。能不能每次输入完命令之后，就自动打开创建好的文件呢？别说，还真能，而且比想象中还要简单。

    其实一开始我还想自己写个小程序把Hexo命令和打开文件的命令结合起来，创建博客的时候就运行这个程序。不过我在其它博客上找到了一个更方便的解决方案。

    原理

    其实这个原理并不复杂，也不需要专业的知识。只需要知道，这种方法是利用Node.js的事件监听机制实现监听Hexo的new事件来启动编辑器，完成自动启动编辑器的操作就可以了。

    没看懂也没关系（因为我也不是很懂），只要照着步骤做就可以了。据说这个解决方案在Hexo的GitHub项目的Issues中也能找到，所以不用担心什么安全问题。

    步骤

    步骤也非常简单。

    1. 在博客的根目录下的scripts目录中新建一个后缀名为.js的JavaScript脚本文件，如果没有这个目录，就新建一个，注意目录的大小写。这个文件可以任意取名，比如auto open.js。

    2. 用文本编辑器打开这个文件。Windows平台输入如下内容：

      var spawn = require('child_process').exec;

// Hexo 2.x 用户复制这段
hexo.on('new', function(path){
  spawn('start "markdown编辑器绝对路径.exe" ' + path);
});

// Hexo 3 用户复制这段
hexo.on('new', function(data){
  spawn('start "markdown编辑器绝对路径.exe" ' + data.path);
});

      Mac平台输入如下内容：

      var exec = require('child_process').exec;

// Hexo 2.x 用户复制这段
hexo.on('new', function(path){
    exec('open -a "markdown编辑器绝对路径.app" ' + path);
});
// Hexo 3 用户复制这段
hexo.on('new', function(data){
    exec('open -a "markdown编辑器绝对路径.app" ' + data.path);
});

      比如我是Windows，Hexo 3，编辑器是Typora，我的文件内容就是这样：

      var exec = require('child_process').exec;
// Hexo 3
hexo.on('new', function(data){
    exec('start "C:/Program Files/Typora/bin/typora.exe" ' + data.path);
});

    3. 保存退出即可。现在通过这个脚本，我们就能监听hexo new这个动作，并且在动作完成之后自动打开编辑器了。

    4. 使用hexo new创建一篇博客，是不是新建的md文件自动打开了？~

    关于scripts目录的解释

    Hexo 有强大的插件系统，使您能轻松扩展功能而不用修改核心模块的源码。在 Hexo 中有两种形式的插件（注：另一种是真的插件（Packages））：

    脚本（Scripts）

    如果您的代码很简单，建议您编写脚本，您只需要把 JavaScript 文件放到 scripts 文件夹，在启动时就会自动载入。

    ——Hexo官网
