NexT主题的Bootstrap Callout标签简介

发表于 2019-06-29 更新于 2021-11-06 本文字数： 1.9k 阅读时长 ≈ 3 分钟

在Hexo博客里想要重点标示一段文字有很多种方式。

比如可以这样

或者这样

代码：

`比如可以这样`
> 或者这样

这些方法是Markdown自带的。不过下面要介绍的方法并不是Markdown的基本语法，而是NexT主题特有的扩充语法。

什么是标签插件

首先来简单了解一下Hexo提供了什么额外的显示格式。

为了扩充Markdown的表现能力（也许），Hexo提出了“标签插件”的概念，根据官方文档的说法，标签插件是用于在文章中快速插入特定内容的插件，比如引言、代码等内容，虽然有些内容Markdown自身的语法就有，不过Hexo提供了另外一种插入它们这些内容的解决方案。

「标签」(Tag Plugin) 是 Hexo 提供的一种快速生成特定内容的方式。 例如，在标准 Markdown 语法中，我们无法指定图片的大小。这种情景，我们即可使用标签来解决。 Hexo 内置来许多标签来帮助写作者可以更快的书写，完整的标签列表可以参考 Hexo 官网。 另外，Hexo 也开放来接口给主题，使主题有可能提供给写作者更简便的写作方法。

内置标签NexT 使用文档

因为Hexo开放了接口，所以NexT主题也有自己的标签插件，其中一种，就是接下来要介绍的Bootstrap Callout，接下来来看看它的显示效果是怎样的。

什么是Bootstrap Callout

Bootstrap Callout就是NexT主题内置的一种标签格式，你可以使用它来标示出你想突出的文字，它的使用方式和效果如下所示：

{% note [class_name] [no-icon] %}
这里写你需要写的内容
{% endnote %}

// 注意上面的class_name和no-icon属性是可以选择的
[no-icon] : Disable icon in note.

其中，class_name可以是以下列表中的一个值：

  • 不填

    Content (without define class style).

  • default

    Content (default style).

  • primary

    Content (primary style).

  • success

    Content (success style).

  • info

    Content (info style).

  • warning

    Content (warning style).

  • danger

    Content (danger style).

效果在编辑文档的时候是看不到的，只有发布到博客上之后才能看见。

更改样式

样式的更改在主题目录下的_config.yml文件里（不是根目录下那个）。找到下面的部分，按需修改即可。

# Note tag (bs-callout)
note:
  # Note tag style values:
  #  - simple    bs-callout old alert style. Default.
  #  - modern    bs-callout new (v2-v3) alert style.
  #  - flat      flat callout style with background, like on Mozilla or StackOverflow.
  #  - disabled  disable all CSS styles import of note tag.
  # style: simple
  style: modern
  # icons: false
  icons: true
  border_radius: 3
  # Offset lighter of background in % for modern and flat styles (modern: -12 | 12; flat: -18 | 6).
  # Offset also applied to label tag variables. This option can work with disabled note tag.
  light_bg_offset: 0

踩坑

  1. Bootstrap Callout标签的格式一定要特别注意，不能把{ % note %}、内容和{ % endnote %}放在同一行，应该每一部分单独成行，否则会出现渲染上的问题（比如和Markdown的语法冲突）。举个例子：

    1. 这样写是错的：

      {% note %}Content{% endnote %}

    2. 这样写也是错误的：

      {% note %}Content
{% endnote %}

  2. 标签的内容部分目前似乎并不支持Markdown格式，如果真的需要可以用HTML的标签来替代（比如# 标题用<h1>标题</h1>代替）。
