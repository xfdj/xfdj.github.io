优化博客（十三）：将博客同时部署到码云上

发表于 2020-02-23 更新于 2021-10-10 本文字数： 1.2k 阅读时长 ≈ 2 分钟

起因

长话短说，原因很简单，就是Github在国内的速度不够快。我感觉我的博客并不花里胡哨，因此打开速度最好也快一点。我不想等博客上所有内容和图标以及卡哇伊的看板娘半天才加载出来，也不想点击搜索按钮搜博文的时候要等老半天，完全没有秒搜的快感。因此，在查看了一些资料之后，我选择同时也将博客部署到国内的代码平台——码云上。这样，相当于博客拥有了两个网址

部署步骤

新建仓库

注册了码云的账号之后，如下图所示新建一个仓库。

图片中的“使用Readme文件初始化这个仓库”复选框要勾选！否则发布博客的时候可能会出现“failed to push some refs to ...”的问题。

图片中的仓库名称一定要填成和自己的个性地址同名！不然的话生成的博客会带二级目录，与\_config.yml的不一致会不导致加载CSS，或者出现404。个性地址可以在仓库网址上看到。后续仓库名称也可以在管理中修改。我第一次图片里就填成了blog，导致出现了问题。

创建完后，如下图所示复制一下仓库的地址（注意不是复制网址）。

配置站点配置文件_config.yml

在博客根目录下打开站点配置文件_config.yml，找到deploy的部分，修改成如下所示的内容。

deploy:
  type: git
  repo: 
    github: GitHub上的仓库地址
    gitee: 刚才复制的仓库地址
  branch: master

如果是之后修改了仓库名称，需要重新复制仓库的地址填到这里。

发布一遍博客

注意在进行下一步之前一定要先做这一步！

用命令hexo d发布一遍博客。第一次发布需要填写用户名和密码，之后Windows会以凭据的形式保存你的用户名和密码。

Gitee Pages设置

接下来我们需要将仓库设置成Gitee Pages服务，类似于Github Pages。设置方法如下图所示。

在服务中选择Gitee Pages选项。

启动之后留意一下显示出来的网站地址，那就是码云上的博客网址啦。再次注意最好将仓库名称改成个性地址同名，这样博客网址不会出现二级目录，会少很多问题。

访问新的博客地址

如果忘记博客地址的话，可以在上一步中的服务页面找到，点进去看看新的博客地址吧！

我的博客的码云地址：https://xfdj_dragon.gitee.io
