什么是CSS

Cascading Style Sheet 层叠样式表

CSS：表现（美化网页）

字体，颜色，边距，高度，宽度，背景图片，网页定位，网页浮动……

CSS的优势

  1. 内容和表现分离
  2. 网页结构表现统一，可以实现复用
  3. 样式十分丰富
  4. 建议使用独立于HTML的CSS文件
  5. 利用SEO，容易被搜索引擎收录

层次选择器

  • 后代选择器：在某个元素的后面

    /* 后代选择器 */
body p {
    background: red;
}

  • 子选择器：一代

    /* 子选择器 */
body>p {
    background: blue;
}

  • 相邻兄弟选择器：向下相邻，弟弟选择器

    /* 相邻兄弟选择器 */
.active+p {
    background: brown;
}

  • 通用兄弟选择器：所有弟弟选择器

    /* 通用兄弟选择器 */
.active~p {
    background: yellow;
}
<!DOCTYPE html>

结构伪类选择器

<!DOCTYPE html>
<html lang="en">

<head>
    <meta charset="UTF-8">
    <title>Document</title>
    <!-- 避免使用class、id选择器 -->
    <style>
        /* ul的第一个子元素 */
        ul li:first-child {
            background: green;
        }
        /* ul的最后一个子元素 */
        ul li:last-child {
            background: red;
        }
        /* 选中p1 */
        /* 选择当前元素的父级元素的第一个子元素，并且只有当子元素为p时才生效 */
        p:nth-child(1) {
            background: yellow;
        }
        /* 选择当前元素的父级元素的第一个类型为p的子元素 */
        p:nth-of-type(1) {
            background: orange;
        }
    </style>
</head>

<body>
    <h1>h1</h1>
    <p>p1</p>
    <p>p2</p>
    <p>p3</p>
    <ul>
        <li>li1</li>
        <li>li2</li>
        <li>li3</li>
    </ul>
</body>

</html>

属性选择器

<!DOCTYPE html>
<html lang="en">

<head>
    <meta charset="UTF-8">
    <title>Document</title>

    <style>
        .demo a {
            float: left;
            display: block;
            height: 50px;
            width: 50px;
            border-radius: 10px;
            background: blue;
            text-align: center;
            color: grey;
            text-decoration: none;
            margin-right: 5px;
            font: bold 20px/50px Arial;
        }

        /* 属性名，属性名=属性值（正则） */
        /*
        = 严格等于
        *= 包含
        ^= 以开头
        $= 以结尾
        */
        /* 存在ID属性的元素 */
        a[id] {
            background: yellow;
        }
        /* ID为first的元素 */
        a[id=first] {
            background: yellowgreen;
        }
        /* class中有links的元素 */
        a[class*="links"] {
            background: green;
        }
        /* herf中以http开头的元素 */
        a[href^="http"] {
            background: red;
        }
        a[href$="pdf"] {
            background: darkred;
        }
    </style>
</head>

<body>
    <p class="demo">
        <a href="https://www.baidu.com" class="links item first" id="first">1</a>
        <a href="" class="links item active" target="_blank" title="test">2</a>

        <a href="images/123.png" class="links item">4</a>
        <a href="images/123.jpg" class="links item">5</a>
        <a href="abc" class="links item">6</a>
        <a href="/a.pdf" class="links item">7</a>
        <a href="/abc.pdf" class="links item">8</a>

浮动

标准文档流

指的是元素排版布局过程中，元素会默认自动从左往右，从上往下的流式排列方式。

块级元素：独占一行

行内元素：不独占一行

display

/*
block 块元素
inline 行内元素
inline-block 行内元素，但是保留块元素的特性
none 消失
*/
div {
    width: 100px;
    height: 100px;
    border: 1px solid red;
    display: none;
}
span {
    width: 100px;
    height: 100px;
    border: 1px solid red;
    display: inline-block;
}

父级边框塌陷的问题

/*
clear 指定一个元素是否必须移动到在它之前的浮动元素下面。
	left 左侧不允许有浮动元素
	right 右侧不允许有浮动元素
	both 两侧不允许有浮动元素
	none
*/

  • 父级元素设置高度

    #father {
    border: 1px solid black;
    height: 500px;
}

  • 父级元素中，末尾增加一个空div标签，清除浮动

    div {
    clear: both;
    margin: 0px;
    padding: 0px;
}

  • overflow

    /*
overflow
	hidden 隐藏
	scroll 滚动条
*/
#content {
    width: 200px;
    height: 150px;
    overflow: scroll;
}
/*
在父级元素中增加一个overflow: hidden
*/
#father {
    overflow: hidden;
}

  • 父类添加一个伪类（推荐）

    #father:after {
    content: '';
    display: block;
    clear: both;
}

定位

相对定位

/*
相对定位：相对于自己原来的位置进行偏移，仍然在标准文档流中。
*/
body {
    padding: 20px;
}
div {
    margin: 10px;
    padding: 5px;
    font-size: 12px;
    line-height: 25px;
}
#father {
    border: 1px solid #666;
    padding: 0px;
}
#first {
    background-color: #a13d30;
    border: 1px dashed #b27530;
    position: relative; /* 相对定位：上下左右 */
    top: -20px;
    left: 20px;
}
#second {
    background-color: #255099;
    border: 1px dashed #255066;
}
#third {
    background-color: #1c6699;
    border: 1px dashed #1c6615;
    position: relative;
    bottom: 10px;
}

绝对定位

没有父级元素定位的情况下，基于浏览器定位。

父级元素存在定位的情况下，相对于父级元素进行偏移。

在父级元素范围内移动。

不在标准文档中，原来的位置不会被保留。

div {
    margin: 10px;
    padding: 5px;
    font-size: 12px;
    line-height: 25px;
}
#father {
    border: 1px solid #666;
    padding: 0px;
    position: relative;
}
#first {
    background-color: #a13d30;
    border: 1px dashed #b27530;
}
#second {
    background-color: #255099;
    border: 1px dashed #255066;
    position: absolute;
    top: 0px;
    left: -10px;
}
#third {
    background-color: #1c6699;
    border: 1px dashed #1c6615;
}

CSS预处理器

CSS是一门标记语言，不是编程语言，因此不可以自定义变量，不可以引用，不具备任何语法支持。它的主要缺陷如下：

  • 语法不够强大，比如无法嵌套书写，导致模块化开发中需要书写很多重复的选择器。
  • 没有变量和合理的样式复用机制，使得逻辑上相关的属性值必须以字面量的形式重复输出，导致难以维护。

CSS预处理器定义了一种新的语言，基本思想是，用一种专门的编程语言，进行Web页面样式设计，再通过编译器转化为正常的CSS文件。常用的CSS预处理器有：

  • SASS：基于Ruby，通过服务端处理，功能强大，解析效率高，需要学习Ruby语言，上手难度高于LESS。
  • LESS：基于NodeJS，通过客户端处理，使用简单。功能比SASS简单，解析效率低于SASS，但在实际开发中足够，建议使用。
