优化博客（一）：解决LaTeX无法显示的问题、更换主题

发表于 2018-09-04 更新于 2021-11-06 本文字数： 1.6k 阅读时长 ≈ 3 分钟

写博客的时候遇到了一些问题，在本地可以渲染的LaTeX语法，到了网上就无法显示了。就像下面这个样子。

本地渲染正常。

网上渲染不正常。

网上有通过安装hexo-math包来解决的文章，但是自己尝试后由于npm出了毛病，并没有成功。

网上的数学公式大多是用MathJax（适用于所有浏览器的数学JavaScript显示引擎——MathJax官网）来渲染显示的。后来听说NexT主题已经集成了MathJax，我最终打算通过安装NexT主题来解决这个问题。

安装NexT主题

在博客根目录下执行如下命令：

git clone https://github.com/next-theme/hexo-theme-next themes/next

意思是用Git把next主题克隆到themes文件夹下。这样安装的好处是之后的更新比较方便。更新时在themes/next文件夹里使用如下命令：

git pull

Git就会拉取最新的分支到本地。

也可以使用子模块功能：

git submodule add https://github.com/next-theme/hexo-theme-next themes/next

随后到博客根目录下打开_config.yml，找到theme属性，并将默认的landscape改为next。

之后清理缓存：

hexo clean

并像之前一样部署，就可以在网站上看到新的主题了。

配置MathJax

主题更换完了，但是MathJax默认没有打开，要更改配置文件来打开它。

进入themes/next目录，找到目录下的_config.yml文件（注意不是根目录下的那个！），打开它，定位到math，把enable的值由默认的false改为true，这样MathJax就打开了，也可以将下面的per_page选项改为false，如果这个值为true，那么MathJax只会渲染在文章开头打了特定标签的文章。如下图所示：

重新部署博客，公式终于显示出来了。

但是新的问题出现了，有部分公式无法正常显示。查阅网上资料得知是由于MathJax与Markdown默认渲染冲突。

更换Hexo的Markdown引擎

尝试了修改Hexo渲染源码，也就是修改nodes_modules/marked/lib/marked.js文件的方法失效后，我选择了另外一种方法，就是把Hexo默认的Markdown渲染引擎hexo-renderer-marked换成了hexo-renderer-pandoc。也有教程建议换成hexo-renderer-kramed或者hexo-renderer-markdown-it的，个人认为还是hexo-renderer-pandoc更好一些，基本上完美解决问题，其它的都还存在部分问题。

安装Pandoc

首先需要安装Pandoc，因为hexo-renderer-pandoc是借助Pandoc来渲染Markdown的。Pandoc是一个类似于格式工厂一样的工具，专门转换标记语言。前往Pandoc官网

安装过程比较简单，基本上一路默认就行了。

更换引擎

就是卸载默认的，再安装新的：

npm uninstall --save hexo-renderer-marked
npm install --save hexo-renderer-pandoc

如果出现了npm ERR!的问题，可以改用Yarn来代替npm，具体更改方式见这一篇文章使用Node.js包管理工具Yarn。如果用Yarn，命令就变成：

yarn remove hexo-renderer-marked
yarn add hexo-renderer-pandoc

hexo-renderer-pandoc有个副作用，就是原来文章里的图片是不会显示图片说明文字的，不过这个插件把说明文字放在图片下面（就像上面的图片一样），影响不是很大，今后看看有没有什么办法去掉了。
