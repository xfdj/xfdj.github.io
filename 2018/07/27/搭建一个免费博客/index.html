搭建一个免费博客

发表于 2018-07-27 更新于 2021-03-15 本文字数： 6.9k 阅读时长 ≈ 12 分钟

我为什么需要一个博客

这个博客的诞生，源于我一直以来的一个想法。

在童年没有什么特殊要求的时候，我使用QQ空间记录日志。

上了大学以后，由于课程的要求，我开通了CSDN的账号撰写技术博客。

这两段经历告诉了我使用第三方博客工具存在的限制：

  1. 如Word一般混乱的排版规则（QQ空间）。QQ空间的文章可以选择纯文本编辑或者HTML，像我这样平凡的小朋友肯定是选择纯文本编辑的，字号、字体、颜色之类的格式用顶端提供的样式条来解决。最后发表出来的文章没有统一的层次结构，看起来混乱且没有美感，配合着QQ空间屎一样的皮肤，让人没有想读下去的欲望，同时显得很low。
  2. 无法消除的广告（QQ空间、CSDN）。使用第三方博客工具躲不掉地要做出些妥协。那些在文章两侧，虽不影响阅读但很影响心情的广告，就像顽固的牛皮藓一般死死贴在那里。
  3. 令人气恼的审核（CSDN）。如果我写了一篇文章，我肯定希望马上能显示在博客上供自己阅读。但是使用CSDN发表博客的时候，时不时的文章就会被拿去审核，无法立刻放出来，我就想问只是几篇技术博客，又不是反动的政治言论，有任何审核的必要吗？

就是说，我在有自己的博客前，已经在用这些现成的产品记录文字了，但是时间一长，现成的产品暴露出这样那样的不方便的时候，我就开始憧憬一个只属于我自己的，几乎没有什么限制的博客。这是客观原因。

当然，还有一些更主观的原因。我曾经向我的好朋友诉苦，说自己在大学里并不快乐。两年的大学生活依然没能让我适应大学的教学风格，也许是自己曾经有过太好的高中老师，我对于一个老师的评价标准可以说到了苛刻的地步。同时，大学的种种也没有自己想象中的美好，没有那么多可以一起讨论学术的同学，总是打游戏的“室友”，只能称之为“PPT复读机”的大学教授，只会罗列术语的教材，很水的课程。我每天没有学到了新东西的快乐，只有深深的孤独。

所以，我需要按照自己的步调，自己的思维来学习，我需要一个整理的空间，一个交谈的对象。可能，会很困难，会付出很多很多的精力，很多很多的时间，可能会拖累学习的进度。但是，我受够了垃圾老师和垃圾教材，我想写一种，只给自己看的，不买弄术语，不用英文缩写装逼，不晦涩不难懂，真正为了学懂学明白的文章。我想证明，大学里有一些知识的门槛是被一些傻瓜人为提高的，难学只是因为讲授的人或书的表达能力捉急而已。于是，这个想法蹦进了我的脑袋里：开一个自己的播客吧。

嗯，首先，得有一个博客。

选择建立博客的路线

作为一个基本不会HTML，不会PHP，不会建站的小白，我在迈出第一步之前相当谨慎，尽量确保我明白自己在干什么，而不是跟着教程瞎做。我查找了很多不同的文章，看到了几种不同的搭建思路。我相信如果大家也去搜索的话，大致分为下面列出的两条路线。这里列出的东西没有必要都了解。

  • 动态博客路线。以WordPress、Ghost为代表。
  • 静态博客路线。以Hexo、Jekyll为代表。更多见11个最流行的静态(博客)网站生成工具。

当然，如果不想不明不白就拥有了一个博客的话，你需要知道我在说什么，也就是动态博客和静态博客的意思以及区别。

  • 动态博客需要服务器和数据库，静态博客不需要。
  • 资源占用上，静态的相比动态占用服务器资源少，还可以托管在GitHub上。
  • 发布更新操作上，由于静态博客没有管理后台，所以发布更新内容要比动态博客繁琐。
  • 访问速度上，由于静态博客没有数据库，所以访问速度更快。
  • 安全性上，静态博客相比动态博客免疫了很多Web攻击套路。

包括一些没有列出来的原因，我最终选择了静态博客。最根本的原因嘛……对于我一个穷逼来说，免费才是第一吸引力啊！

我的最初方案：

  • 博客框架：Hexo。这是一个静态博客框架。至于博客框架是什么意思，读到怎么发布一篇博客的时候就能明白了。
  • 托管平台：GitHub。这是世界上最大的同性交友网站。对于我们来说，只要理解成是博客网站保存的地方就可以了。

这么选择的理由主要有这么几点：

  • GitHub+Hexo的组合是比较流行的博客解决方案，在网上有丰富的资料可以参考。
  • 博客属于内容导向型的网站，静态博客就可以很好的达到我的目的。
  • GitHub上搭建的博客保存可靠，空间足够。

我暂时没有追求（不排除以后会有追求的可能）：

  • 自己的域名。目前来说，只要能访问得到我的博客就足够了。
  • 个性化自己的博客。至少这篇博文的目标只是把博客搭起来，后续的优化会在其他文章里呈现。

开始搭建

诚实的说，这里的搭建过程和网上大多数你可以搜得到的教程大同小异。

环境的准备

环境，就好像一间可以满足你的办公需求的办公室。所以搭建博客的第一步，就是搭建好所谓的环境。我的操作系统是Windows 10，以下的环境搭建全部都是在Windows 10中完成。

安装Node.js

Node.js 是一个 JavaScript 运行环境。

Node.js 的包管理器 npm，是全球最大的开源库生态系统。

——Node.js中文网

为什么需要Node.js呢？因为Hexo需要通过Node.js的包管理器npm下载，Hexo的运行也需要Node.js的支持。最好下载LTS版本，也就是长期支持版。前往官网下载Node.js

安装过程很简单，一路默认就行了。

👆点击Next。

👆勾选I accept the terms in the License Agreement，点击Next。

👆选择安装路径，点击Next。

👆选择安装组件，Node.js runtime和npm package manager一定要装，Add to PATH也选了，不然接下来在命令行输入指令会没反应，点击Next。

👆点击Install安装。

👆点击Finish完成安装。

安装Git

Git is a free and open source distributed version control system designed to handle projects.

Git是一个免费开源的分布式版本管理系统，被用来管理项目。

——Git官网

你可以把Git当成是拿着毛笔记录历史的史官，你在他的注视下做的一切改动都会被忠实地记录下来，并且厉害的是，这个史官可以恢复你所做的改动。

不明白版本管理系统是什么也没关系，安装它是因为把博客发布到GitHub上需要通过Git的命令来完成。当然Git本身是个很值得花点时间掌握的工具，网上有数不清的教程，这里就不赘述了。

安装步骤比较多，但是很简单，如果看不明白选项是什么意思就按照默认设置，不影响接下来的使用。

👆点击Next。

👆选择安装路径，点击Next。

👆选择安装组件，按照图示勾选即可，点击Next。

👆创建开始菜单，按照默认就行，点击Next。

👆选择默认编辑器，这个不影响，之后也是可以用自己喜欢的编辑器的，可以直接点击Next。

👆选择第二个可以通过添加环境变量的方式，在Windows的命令行中使用Git，点击Next。

👆选择HTTPS传输后端，我们用OpenSSL库，点击Next。

👆配置行结束转换。Windows平台和Unix/Linux平台的行结束符不一样，在Windows下选第一个，可以让Git在提交的时候自动换成Unix格式的结束符。点击Next。

👆配置终端模拟器以与Git Bash一起使用，选择第一个比较方便，点击Next。

👆配置额外的选项，按照默认，点击Install进行安装。

👆安装完成，点击Finish结束安装。

安装Hexo

打开命令提示符或者Git Bash，输入下面的命令：

npm install -g hexo-cli

这条命令会调用Node.js的包管理器npm安装Hexo。-g看帮助信息是全局安装的意思，hexo-cli是Hexo的命令行界面。

输入完命令后稍微等待一下，如果结果类似上面，有+ hexo-cli的字样，就说明安装成功了，npm给出的WARN信息可以忽略。

创建本地博客目录

找一个自己喜欢的目录位置，我们先在本地创建Hexo的博客目录。比如说我在桌面上新建了一个叫blog的文件夹，进入这个文件夹下，右键打开Git Bash（当然你也可以用命令提示符，只是注意路径要改到这个目录）。

输入下面的命令：

hexo init

这条命令在当前的目录下搭建Hexo的框架。

如果最后出现了INFO Start blogging with Hexo!，证明初始化成功完成。

接着执行下面几行命令，先不用管什么意思，很快你就会明白的，我们先来看看成果。

hexo generate
hexo server

注意这些指令都要在刚才创建博客的目录下运行。

之后我们打开浏览器，输入网址http://localhost:4000/，就可以看到Hexo默认的博客网页了。

可以看到博客上有一篇自动生成的文章，告诉你怎么在这个博客上发文章，可以读一读。

至此，本地的配置基本上完成了。接下来是网络上的配置。

配置GitHub

还记得Git是什么吗？Git是一个史官版本管理工具，既可以在本地管理，也可以在提供服务的网站上管理，而GitHub就是一个那样的网站。

为什么需要GitHub呢？因为保存在自己电脑上的博客，是不能在其他电脑上访问的。为了让你的博客不管在哪台电脑上都能看到，我们需要把博客网站“放”到另一个网站上，“放”到一个有网的电脑都能访问的网站上，所以我们需要GitHub。

如果你没有GitHub账号，那么需要注册一个。前往GitHub官网

注册完账号之后登录进去，点击右上角的➕，然后点击New repository，意思是新建仓库。

之后会弹出一个完善信息的页面，在Repository name框内填入你的Owner名称.github.io，比如我需要填的是xfdj.github.io，一定要按照格式来，这样GitHub才会把它当做一个网站来解析。之后直接点击Create repository就创建成功了。

配置Git

因为Git的发布到GitHub上时需要名称以及邮箱信息，所以当有了账号之后需要在本地配置一下Git，打开Git Bash，输入下面两行命令：

git config --global user.name "你的Owner名称"
git config --global user.email "你注册时的邮箱"

如果没有什么输出的话就证明没问题了。

配置Hexo参数

为了让本地的Hexo目录与在GitHub上新建的仓库关联起来，我们需要配置一下Hexo。

现在需要进入到博客的目录下，找到名为_config.yml的文件，用默认或其它文本编辑器打开，以修改其中的参数信息。

来说一下需要修改的内容。

注意：修改时使用英文标点符号，冒号后面一定要有一个空格，不填的内容直接空着就好。

# Site
title: xfdj的博客
subtitle:
description:
keyword:
author: xfdj
language: zh-CN
timezone: Asia/Shanghai

首先是网站的信息，上面放出的是我的博客的信息，从上到下依次填的是：

  • title 网站标题，会显示在浏览器的标签上。
  • subtitle 副标题，可以不填
  • description 网站描述，可以不填
  • keyword 网站关键词，一般和description用于SEO（搜索引擎优化），可以不填
  • author 你的名字
  • language 网站使用的语言，一般填zh-CN。
  • timezone 网站时区，默认是电脑时间

language和timezone都是有输入规范的，详细可参考语言规范和时区规范。

# URL
## If your site is put in a subdirectory, set url as 'http://yoursite.com/child' and root as '/child/'
url: https://xfdj.github.io
root: /
permalink: :year/:month/:day/:title/
permalink_defaults:

接着配置url部分，把url后面的网址改成你的博客网址，就是你在新建仓库时填写的你的Owner名称.github.io，其它的保持默认就好。（这一部分是可选的，不配置不会有影响，最新的建议是用#注释掉url和root行，这样在同时部署到多个网站时会减少困惑的发生）

# Deployment
## Docs: https://hexo.io/docs/deployment.html
deploy:
  type: git
  repo: https://github.com/xfdj/xfdj.github.io.git
  branch: master

最后是最关键的部署部分的配置，你需要在type后填写git，然后再加2项：repo和branch，在repo处填写一串地址，你可以进入刚刚创建的仓库，在下图所示的地方找到地址，点击右侧的按钮即可复制下来，粘贴到repo处。branch后面填写master，先不必理解这是什么意思。注意repo和branch要缩进，和type对齐。

配置完之后记得保存。

撰写博文

如果你阅读了Hexo默认的第一篇文章，就知道如何撰写并发表博文了。

新建文章

在博客目录下（注意，不是在别的目录下）打开Git Bash或者命令提示符，执行下列命令创建一篇新文章。标题要用英文双引号括起来。

hexo new "你的文章标题"

比如我新建了一个名为test的文章，如果出现了下图所示的类似输出，就说明文章已经新建成功了。

然后就可以到INFO给出的路径下打开这个名为test.md的文件写文章了。

编辑文章

写文章的格式是Markdown，这是一种非常简单好学的轻量级的标记语言，网上有大量的教程，不需要多长时间就能使用，这里就不介绍了。

下图是示范性在test.md文件里写了一句话。1至5行是自动生成的，之后的文章中会介绍如何编辑这一部分，目前直接忽略就行了。

预览并发表文章

文章写好之后，我们依然可以用之前用过的三条命令来预览和发表文章。

hexo generate

之前没有解释，这条指令的意思是生成静态页面至public目录，保存了文章之后要执行这条命令Hexo才会把Markdown文本转换成网页，最后推送到GitHub上的只有public目录中的网页内容。在这条命令之前你可以先执行hexo clean命令来清除缓存和原有的public目录。当然一般情况下也可以不执行。

hexo server

你依然可以通过这条命令在本地预览博客的样子，像之前一样在浏览器打开http://localhost:4000/即可。这不是必须的，你也可以跳过。

hexo deploy

最后一步，发布到GitHub上，只需要输入这一句，你很可能会看到这个结果。

这是因为没有安装Git的部署插件，我们需要输入另外一条命令：

npm install --save hexo-deployer-git

这条命令会在Hexo上安装Git的部署插件，--save表示将安装包信息加入到dependencies（生产阶段的依赖），有什么用我也不太懂，不过大家的教程都把这个参数加上了。

补：--save和-g的区别是前者只是把包安装在该文件夹下，并写入package.json，后者是安装在全局，并且会加入环境变量，所有文件夹下都可以使用。package.json的作用可以参考之后的文章将博客迁移到另一台电脑。

同样的，出现了+ hexo-deployer-git的信息就表示安装成功。重新执行hexo deploy（最好是重新打开Git Bash窗口），在一堆乱七八糟的消息之后会弹出一个窗口，输入你的GitHub用户名和密码，等待最后出现INFO Deploy done:的信息之后，在浏览器打开你的Owner名称.github.io的网址，是不是能看到和本地预览一样的网页？如果是的话，恭喜你，你可以在任何一个有网的地方通过这个网址访问到你的博客了！

命令总结

总结起来，博客发布只需要记住下面的几条指令（#后面是注释）：

hexo new "name"   # 新建文章
hexo clean        # 清除缓存
hexo generate     # 生成页面
hexo server       # 本地预览
hexo deploy       # 部署

至此，一个有基本功能的简陋博客已经搭建起来了，之后我会更新博客的优化以及博客迁移的内容。
