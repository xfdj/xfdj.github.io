将博客迁移到另一台电脑

发表于 2018-08-21 更新于 2022-05-15 本文字数： 1.6k 阅读时长 ≈ 3 分钟

注：为了避免发生问题，本篇教程只适用于依照我之前的教程搭建的博客。

迁移博客的原因

需要重装电脑，所以博客的迁移自然成了一个问题。由于Hexo的配置以及文章均保存在本地，所以暂时没办法像其他在线博客平台一样随时撰文发布。要想在新的电脑上继续更新原来的博客，就需要进行迁移。

迁移博客的要求

首先，我们想让之前的配置在新的电脑上依然有效，模板、皮肤以及各种其他的设置都能保留下来，而不用重新配置。

其次，博客之前的文章不能丢，并且发布的时间顺序以及其他信息要保持原样。

最后，博客要可以继续发布在原来的GitHub仓库上，新的博文可以通过之前的网址访问到。

开始迁移

网上这方面的资料不是很多，一开始查询到的解决方案各执一词。经过对比之后，我打算参考以下两篇文章的方案：

hexo博客迁移到另一台电脑 - CSDN博客

换了电脑如何使用hexo继续写博客 - 明志健致远 - 博客园

复制原博客文件夹的文件

如果你是像我一样重装了电脑，那么在重装之前你需要先复制原博客文件夹中的这些项（以/结尾的表示是目录）：

文件和目录 用途
_config.yml 站点配置文件，很多全局配置都存在这里
package.json 里面存着Hexo框架的参数（比如版本号什么的）和依赖的插件
package-lock.json（可选择不复制） 我发现自己的文件夹下有这么一个文件，查询了一下发现package.json只能锁定依赖插件的大版本，而这个文件可以锁定小版本，这就是lock的意思，它可以保证依赖一致。
yarn.lock（如果有的话） 安装了Yarn之后产生的文件，作用和package.json类似
scripts/（如果有的话） 用来存放脚本代码的文件夹
scaffolds/ 字面意思是“脚手架”，里面存着模板文件（像post.md什么的），当你新建一篇文章的时候，Hexo实际上会把相应的模板文件（默认是post.md）复制到source文件夹下的_posts文件夹里，然后修改文件名。
source/ 最重要，因为这是存放博客文章的地方，没了它，你写的文章就都没了。
themes/ 博客网站的主题存在这里。
.gitignore 用于声明被Git忽略的文件。

无需拷贝的部分有：

文件和目录 用途
node_modules/ 里面存放着npm安装的依赖包，这些包的信息都保存在package.json和package-lock.json里了，不需要同步。
.deploy_git/ Hexo默认的.git配置文件夹，不需要同步。
public/ 根据source/的内容自动生成，不需要同步。
db.json 自动生成的数据库文件，不需要同步。

将这些文件复制到U盘或者其他不被影响的存储介质上之后，就可以转战新电脑了。

安装必要软件

首先需要在新电脑上安装并配置Git和Node.js，方法见我之前的文章搭建一个免费博客。

粘贴文件

在你需要建立博客的目录下粘贴原电脑上复制出来的文件和文件夹。

安装Hexo

这个步骤之前漏掉了，因为我之前以为npm会在下一步安装Hexo，结果当我想要写新的博客的时候，就会出现这样的错误：

这时候需要在命令行或Git Bash中输入这个指令来安装Hexo：

npm install -g hexo-cli

这条指令会在全局安装Hexo并且将它加入环境变量，之前的错误就是由于bash在环境变量里找不到Hexo导致的。这条指令更详细的解释参考之前的文章搭建一个免费博客。

安装所需模块

在博客目录下打开命令行或Git Bash，输入如下指令：

npm install

如果安装了yarn可以使用：

yarn

如果出现了上图中added xxx packages in xxxs的字样即为安装成功。

至此，你就可以在新电脑发布博客了，发布过程和之前完全一样。

之后如果有更加方便的迁移方法，我会发表在另外的文章中。

更新主题

如果是使用git的子模块来管理主题的话，使用

git submodule init
git submodule update

两句命令来更新主题。
