修改Ubuntu 18.04的启动效果

发表于 2018-10-01 更新于 2021-09-03 本文字数： 4.7k 阅读时长 ≈ 8 分钟

Ubuntu 18.04的启动效果看起来比较土，所以打算更换Ubuntu的Grub主题和加载动画。在网上查找资料时发现有许多方法都已经过时，所以自己在这里整理出适用于18.04版本的方法。亲测可用。

更改Grub主题

Grub是Linux的系统启动管理器。如果你的电脑上同时安装了Windows和Linux，可以在Grub里选择你要启动的操作系统。Ubuntu 18.04使用的是Grub2作为默认启动管理器。

下载Grub主题

https://www.gnome-look.org/

这个网站有包括Grub主题在内的许多主题免费下载。

选择左侧的Grub Themes链接，就能看到其他人制作的Grub启动器主题，右侧的3 Months Ranking是近三个月的主题下载排名，排名高的一般都是比较好看的主题，可以参考。此外，每个主题有评分和更新日期，尽量选择离现在比较近的那些主题，因为太久之前更新的主题有可能不兼容现在的Grub。

以Grub-theme-vimix这个主题为例。找到Grub-theme-vimix的页面，点击页面中的Files，然后下载。

安装Grub主题

将下载好的文件解压，解压出来的文件夹里面有一个名为Vimix的文件夹和一个名为Install的文件。如下图所示。

我们只需用到文件夹，Install文件是用来自动安装主题的，打开这个文件的内容就能看到Grub主题是怎么被安装的，鉴于有些主题里的文件太老，不适合最新的系统，所以我们采取手动安装的方式。

首先将Vimix文件夹拷贝到/boot/grub/themes目录下（需要root权限，如果没有themes文件夹就新建一个）。用命令行的指令如下（首先进到Vimix文件夹所在的目录）：

sudo cp -r Vimix /boot/grub/themes

-r表示复制目录及其子目录内的所有内容。

第二步是修改Grub的配置文件。找到并打开/etc/default目录下的grub文件（也需要root权限）。在最后一行加上：

GRUB_THEME="/boot/grub/themes/Vimix/theme.txt"

大概的意思就是使用刚才复制到themes目录下的主题，在Vimix文件夹下有一个名为theme.txt的文件，它会告诉Grub如何加载这个主题。

编辑完不要忘记保存。

最后在命令行bash中输入如下命令：

sudo update-grub

这个命令会根据刚才的配置文件更新Grub的配置。重新启动一下，你就能看到刚才安装的主题了。

删除Grub主题

要想删除安装的Grub主题，只需要将对应的主题文件夹（比如Vimix）移除，再把/etc/default/grub里加上的那一行删掉，最后执行sudo update-grub命令更新Grub配置即可。

替换主题背景图片

如果想把主题的背景图片换成自己喜欢的图片，可以查看themes.txt文件，找到desktop-image一项，后面的图片就是背景图片，所以在文件夹中找到图片，用自己准备好的图片代替（名称要完全相同）就行了。

替换主题文字

如果对主体的文字不满意，想要换成自己喜欢的文字，由于Grub不支持.ttf、.otf格式的字体文件，所以准备好字体之后，需要用grub-mkfont生成Grub专用的字体格式：.pf2格式。举个例子，如果准备的字体名称是myfont.ttf，在终端执行：

grub-mkfont -o myfont.pf2 -s 32 -n "myfont" myfont.ttf

-o表示输出的文件名，-s表示字体的大小，-n表示字体的名称（用于修改themes.txt文件），最后是字体文件名。

执行成功后目录下就会多出一个myfont.pf2字体文件，将这个文件复制到主题文件夹和themes.txt同目录下。

接下来打开themes.txt文件，将terminal-font、item_font、font等所有带font的配置都改成之前-n设置的字体名称，比如改成：terminal-font:"myfont"。保存退出。

然后按照之前的安装步骤正常安装即可。

更改Plymouth主题

Plymouth is the application which provides the graphical "splash" screen when booting and shutting down an Ubuntu system.

——Ubuntu Wiki

如Ubuntu Wiki所说，Plymouth就是显示下面这个紫薯效果的东西。

下面就来更换这个默认的主题。

警告：更改Plymouth主题有让电脑变砖的风险，请小心操作并提前备份好你的重要数据!

下载Plymouth主题

Plymoyth的主题同样是到刚才的网站

https://www.gnome-look.org/

下载。点击左侧的Plymouth Themes链接就可以看到许多Plymouth主题，不过和Grub主题一样，有些Plymouth主题由于“年代久远”，并不兼容现在的Plymouth，原则上来说一般不建议下载2010年以前的主题。

Plymouth主题的下载方式和Grub主题的一样，这里就不再赘述。

安装Plymouth主题

为了接下来的操作方便，首先安装一些Plymouth的默认主题。可以通过下面这个命令安装：

sudo apt-get install plymouth-themes

这条命令会自动创建/usr/share/plymouth/themes并且附带几个简单的主题。

下面开始安装自定义主题。以Ubunut Spinner Logo Plymouth这个主题为例（从下载下来的文件来看，这里是网页上的主题标题打错了）。

解压下载好的文件，里面有一个名为ubuntu-spinner-logo的文件夹还有一个readme文件，那个文件夹就是我们需要的主题。

如果你去看Install Read Me文件的话，你会发现这个主题是要复制到/lib/plymouth/themes这里去，但是别着急，刚才说到有一些主题是较早时间发布的老版本主题，因此这些路径是已经失效的，现在Plymouth的主题放置路径是在刚才安装plymouth-themes时提到的那个路径下。

首先，打开ubuntu-spinner-logo目录下的ubuntu-spinner-logo.plymouth文件，文件里两个路径：

刚才说过这两个路径已经失效，因此需要把他们更改过来，也就是将

ImageDir=/lib/plymouth/themes/ubuntu-spinner-logo
ScriptFile=/lib/plymouth/themes/ubuntu-spinner-logo/ubuntu-spinner-logo.script

改成

ImageDir=/usr/share/plymouth/themes/ubuntu-spinner-logo
ScriptFile=/usr/share/plymouth/themes/ubuntu-spinner-logo/ubuntu-spinner-logo.script

并保存。

修改完以后，要顺便检查一下目录名称。从刚才编辑的文件中我们可以得知这个主题的名字叫ubuntu-spinner-logo，Plymouth的主题名称和主题文件名以及主题目录名，必须完全一致，不然会报错。所以检查一下文件夹名是否也为ubuntu-spinner-logo，如果不是要改过来。

接下来将ubuntu-spinner-logo文件夹拷贝到相应的目录下：

sudo cp -r ubuntu-spinner-logo /usr/share/plymouth/

复制完以后，还需要使用下面这条命令安装这个主题：

sudo update-alternatives --install /usr/share/plymouth/themes/default.plymouth default.plymouth /usr/share/plymouth/themes/ubuntu-spinner-logo/ubuntu-spinner-logo.plymouth 100

这里解释一下--install后面的四个参数的含义，第一和第二个参数是默认主题的路径和名称，安装主题时这两个参数都一样，第三个参数是你需要安装的主题的路径，需要填的是对应的主题目录下.plymouth的路径，最后一个参数表示的是优先级，不需要知道是什么意思，都设成100就好。

接着使用下面的命令选择刚刚安装的主题：

sudo update-alternatives --config default.plymouth

如果安装成功了，就会在选项中出现候选的主题，如下图所示。

最后输入一条命令使刚才选择的设置生效：

sudo update-initramfs -u

如果没有问题，重启一下，就可以看到效果了。

可能出现的错误

如果在最后两步不幸遇到了和下面类似的错误提示：

W: plymouth module "(/usr/lib/i386-linux-gnu/plymouth//.so)" missing, skipping that theme.

据说这是一个内核级的错误，这时候千万不要重启，不然可能会卡在开机界面。这种情况有可能是路径设置错误，或者是主题太老，需要删除主题，尝试检查路径并重来一遍。

删除Plymouth主题

和Grub主题不一样，Plymouth主题涉及到安装命令，因此删除主题不是简单地移除文件夹，还需要相应的卸载命令，卸载命令如下（以ubuntu-spinner-）：

sudo update-alternatives --remove default.plymouth /usr/share/plymouth/themes/ubuntu-spinner-logo/ubuntu-spinner-logo.plymouth

然后移除文件夹，用sudo update-alternatives --config default.plymouth命令重新指定主题，最后用sudo update-initramfs -u更新配置。主题才算删除。
