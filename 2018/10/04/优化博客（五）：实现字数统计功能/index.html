优化博客（五）：实现字数统计功能

发表于 2018-10-04 更新于 2022-01-27 本文字数： 1k 阅读时长 ≈ 2 分钟

纯粹是因为闲得慌，才想要添加字数统计的功能hhh。

在网上搜索了许多文章，大部分是安装hexo-wordcount，然后在主题配置下手写配置。个人不太喜欢这种折腾得很复杂的方式，因为一不小心就忘记之前自己是怎么操作的了，改回去也不方便，最后就是对更新的支持比较弱。

在浏览了NexT的_config.yml配置文件之后，我发现8.2.2版本的NexT默认支持的是Hexo Word Counter这款插件，于是打算尝试一下。

注：本文实现基于NexT，不保证其它主题适用。

安装插件

又注：之后的文章中插件管理默认使用Yarn包管理工具，如果不知道Yarn是什么请移步我的这篇博客：使用Node.js包管理工具Yarn

和Hexo的其它插件安装方法一样，在博客根目录打开命令行并输入以下命令安装插件。

yarn add hexo-word-counter

修改站点配置文件

以下配置方法参考自插件的GitHub主页。

进入博客根目录，打开_config.yml文件，在最后加入下面的配置：

# Post wordcount display settings
# Dependencies: https://github.com/theme-next/hexo-symbols-count-time
symbols_count_time:
  symbols: true           # 每篇文章的字数
  time: true              # 每篇文章的阅读时间
  total_symbols: true     # 站点总字数
  total_time: false       # 站点总阅读时间
  exclude_codeblock: true # 不包括代码块里的文字
  awl: 2                  # 词语的平均长度
  wpm: 300                # 每分钟阅读的词数
  suffix: "mins."         # 后缀

将想要显示的选项设置为true，不想显示的设置为false即可。

修改主题配置文件

进入主题目录themes/next，打开_config.yml文件，找到symbols_count_time，调整选项参数。（可以按照最新版NexT推荐的方法修改）

# Post wordcount display settings
# Dependencies: https://github.com/theme-next/hexo-symbols-count-time
symbols_count_time:
  separated_meta: false  # 换行显示
  item_text_post: true   # 显示文章选项的注释
  item_text_total: true  # 显示站点选项的注释
