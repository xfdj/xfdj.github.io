优化博客（六）：添加图片预览

发表于 2018-10-05 更新于 2021-11-06 本文字数： 1k 阅读时长 ≈ 2 分钟

本文基于NexT主题。

NexT主题默认图片是不能预览的，需要在浏览器的新标签页打开查看原图。

在NexT主题的配置文件里发现NexT主题使用一个叫做theme-next-fancybox3的模块实现图片预览功能。以下是配置文件相应的片段：

# Fancybox. There is support for old version 2 and new version 3.
# Please, choose only any one variant, do not need to install both.
# For install 2.x: https://github.com/theme-next/theme-next-fancybox
# For install 3.x: https://github.com/theme-next/theme-next-fancybox3
fancybox: false

于是照着注释的网址，我在GitHub上找到theme-next-fancybox3的页面，并照着官方文档的指示安装。

安装方法和之前的插件有些不一样，因为npm以及Yarn包管理器的源里头并没有收录它。下面是安装方法。

安装模块

下载模块

进入博客根目录，在根目录下执行语句：

git clone https://github.com/theme-next/theme-next-fancybox3 themes/next/source/lib/fancybox

可以看到我们是使用Git来下载theme-next-fancybox3模块的。它会把整个模块从网站上克隆到themes/next/source/lib/fancybox目录下。

更改主题配置文件

打开NexT的主题配置文件_config.yml（注意不是博客根目录下的同名文件），找到上文中提到的相应片段。

将fancybox: false更改为fancybox: true，保存并重新部署博客。

单击博客上图片效果如下：

卸载模块

卸载方法很简单，由于Git的安装模块只是从网页上原封不动地拷贝到目标目录下，所以只需要把themes/next/source/lib/fancybox文件夹删除，再将主题配置文件里的fancybox: true改回fancybox: false即可。

更新模块

如果闲着没事干想更新一下模块的话，只需在themes/next/source/lib/fancybox下打开命令行或Git Bash，执行

git pull

指令即可。Git会自动拉取到最新的更新。
